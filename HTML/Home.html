HyperText Markup Language

Semantic Tag

의미론적 태그

의미 전달

시맨틱 태그는 문서의 구조와 콘텐츠를 더 명확하게 표현할 수 있습니다. 각각의 태그가 특정한 의미를 갖기 때문에, 코드를 읽는 사람이나 검색엔진 등이 쉽게 이해할 수 있습니다.

웹 접근성 향상

시맨틱 태그는 웹 페이지를 보다 접근성 있게 만들어줍니다. 스크린 리더와 같은 보조 기술을 사용하는 사용자는 문서의 구조를 이해하고 내용에 더 쉽게 접근할 수 있습니다.

검색엔진 최적화(SEO)

시맨틱 태그는 검색엔진이 웹 페이지를 분석할 때 페이지의 구조와 내용을 이해하는데 도움을 줍니다. 따라서 올바른 시맨틱 요소를 사용하면 검색엔진 최적화에 도움을 줄 수 있습니다.

코드 유지보수 향상

시맨틱 태그를 사용하면 코드의 가독성이 향상되고, 다른 개발자가 코드를 이해하고 유지보수하는 데 도움이 됩니다. 또한 스타일과 레이아웃을 변경할 때도 더 쉽게 작업할 수 있습니다.

HyperLink

하이퍼링크의 속성

  • href: 링크가 이동할 대상 URL을 지정합니다.
  • target: 링크를 열 위치를 지정합니다. "_blank"는 새 창에서 열도록 지정합니다.
  • title: 링크에 대한 추가 정보를 제공합니다. 마우스를 링크 위로 올리면 툴팁으로 표시됩니다.
  • rel: 링크의 관계를 지정합니다. 예를 들어 "nofollow"는 검색 엔진 로봇에게 해당 링크를 따라가지 말라는 지시를 내립니다.
  • aria-label: 접근성을 위해 스크린 리더 사용자에게 링크의 의미를 설명합니다.

주의사항

  • 검색 엔진은 텍스트를 기반으로 색인화하므로 링크 텍스트에 키워드를 포함하는 것이 좋다.
  • 링크 텍스트는 짧고 간결하게 작성되어야 합니다. 필요한 내용만을 담아 사용자에게 명확한 정보를 제공해야 합니다.
  • 같은 텍스트를 가진 여러 링크는 가독성을 해치므로 피하는 것이 좋습니다.
  • 외부 콘텐츠에 대한 링크인 경우, 다운로드 파일의 크기나 형식과 같은 정보를 포함시켜 사용자에게 미리 알리는 것이 좋습니다.

Structure

  • Header

    <header> 요소는 일반적으로 내비게이션 링크, 로고, 검색 폼 등을 포함하는 소개 콘텐츠를 나타냅니다.

  • Navigation Bar

    <nav> 요소는 내비게이션 링크를 정의하는 컨테이너로 사용됩니다.

  • Main Content

    <main>,<section>,<article> 요소는 문서의 주요 콘텐츠를 나타내며, 헤더, 푸터, 내비게이션을 제외한 콘텐츠를 포함합니다.

  • Side Bar

    <aside> 요소는 메인 콘텐츠 외에 부가적인 콘텐츠를 나타내는데 사용되며, 일반적으로 사이드바로 표현됩니다.

  • Footer

    <footer> 저작권, 연락처등을 나타낸다. 일반적으로 본문과 관계없는 범용적인 내용을 담지만, SEO를 위해 Link를 넣기도 한다.

참고

<div> 요소는 non-Semantic이므로, 아무런 의미가 없습니다. 이러한 이유로 Semantic 태그의 이점을 받지 못하며, 따라서 사용을 지양하는 것이 좋습니다.

Text

Block-level Elements

블록 수준 요소는 한 줄을 차지하고 다음 요소는 새로운 줄에 배치됩니다. 일반적으로 부모 요소의 전체 너비를 차지하며, 블록 수준 요소 내에 다른 블록 수준 요소나 인라인 요소가 올 수 있습니다.

  • <p>: 단락을 나타냅니다.
  • <h1> to <h6>: 제목을 나타냅니다. h1부터 h6까지 있으며, 숫자가 클수록 중요도가 낮습니다.
  • <blockquote>:
    인용문을 감싸는 블럭

Inline Elements

인라인 요소는 문장 안에서 다른 요소와 함께 사용될 수 있으며, 새로운 줄에 시작하지 않습니다. 일반적으로 부모 요소의 너비를 차지하지 않으며, 다른 인라인 요소나 텍스트와 함께 배치됩니다.

주요 인라인 요소:

  • <strong>: 강조 텍스트
  • <em>: Italic 텍스트

  • <cite>: 인용문의 제목
  • <q>: 인용된 텍스트
  • <i>: 이탤릭체
  • <b>: 굵은 텍스트
  • <u>: 밑줄 텍스트
  • <small>: 작은 텍스트
  • <sup>: 윗첨자
  • <sub>: 아래첨자
  • <abbr>: 약어
  • <time>: 시간
  • <del>: 삭제된 텍스트
  • <ins>: 추가된 텍스트

Table

테이블 생성하기

HTML에서 테이블을 생성하는 가장 기본적인 요소는 <table>입니다. 테이블은 행과 열의 직사각형 형태로 데이터를 구성합니다.

테이블은 다음과 같은 요소로 구성됩니다:

  • <table>: 전체 테이블을 정의합니다.
  • <tr>: 각각의 행을 정의합니다.
  • <td>: 각각의 데이터 셀을 정의합니다.
  • <th>: 각각의 헤더 셀을 정의합니다. (<td>와 비슷하지만 헤더 역할을 합니다.)

테이블 유형

테이블은 주로 두 가지 유형으로 사용됩니다:

  • 데이터 테이블: 실제 데이터를 나타내는 데 사용됩니다.
  • 레이아웃 테이블: 웹 페이지의 레이아웃을 구성하는 데 사용됩니다. 이는 CSS를 사용하여 레이아웃을 구성하는 것이 일반적이며, 테이블을 사용하는 것은 권장되지 않습니다.

SEO를 향상시키는 방법

큰 범위에서의 SEO 향상

  • 고품질 콘텐츠 제공

    관련성 높은 주제에 대해 깊이 있는 콘텐츠를 작성하여 정보를 제공하는 동시에 유용한 해결책을 제시합니다.

  • 키워드 최적화

    키워드를 사용자에게 자연스럽게 보이는 위치에 삽입하고, 제목, 메타 설명, 헤딩 태그 등에 키워드를 포함합니다.

  • 링크 구조 최적화

    내부 링크를 적극적으로 활용하여 웹 사이트의 페이지 간 연결성을 강화하고, 사용자 경로를 고려하여 사용자가 필요로 하는 정보로 유도합니다.

  • 모바일 최적화

    반응형 또는 모바일 친화적인 디자인을 구현하고, 모바일 속도 최적화를 위한 이미지 최적화 및 리소스 압축을 수행합니다.

  • 사이트 속도 최적화

    이미지 최적화, CDN(Content Delivery Network) 도입, 캐싱 정책 구현 등을 통해 사이트 속도를 최적화합니다.

  • 메타 데이터 최적화

    각 페이지에 메타 태그를 추가하고, 제목과 설명을 정확하게 작성하여 검색 결과에 표시될 수 있도록 합니다.

  • 사이트맵 제출

    XML 사이트맵을 생성하여 검색 엔진에 제출하여 모든 페이지가 인덱싱되고 검색 결과에 포함될 수 있도록 합니다.

  • 소셜 미디어 활용

    콘텐츠를 소셜 미디어에 공유하기 쉽도록 소셜 미디어 공유 버튼을 추가하고, 소셜 미디어 플랫폼에서 활발한 활동을 합니다.

  • 로컬 SEO

    로컬 비즈니스 디렉토리에 등록하고, 지역 키워드를 타겟팅하는 콘텐츠를 제작하여 로컬 SEO를 강화합니다.

  • 유저 경험 최적화

    사이트 내비게이션의 단순화 및 향상을 통해 사용자 경로를 최적화하고, 페이지 로딩 시간을 단축합니다.
